Presion Nextion Page

Description

Introduction

This document provides an in-depth explanation of a Nextion display script responsible for managing touch components, navigation drawers, navigation bars, main containers, themes, text, and visual elements. The script enables touch controls and sets the visual theme based on user preferences and language selections.

Script Overview

The script comprises the following sections:

  1. Disabling Touch Components (tsw h0,0 and tsw h1,0) - This section disables touch components h0 and h1, preventing user interactions with these elements.

  2. Navigation Drawer - This section defines the visual elements of the navigation drawer, including circles, lines, and fills, using specified colors. The navigation drawer provides a menu interface.

  3. Navigation Bar - This section creates the navigation bar with circles, lines, and fills, using the specified colors. It also divides the navigation bar into individual segments for navigation.

  4. Main Containers - Two main containers are defined, each consisting of circles, lines, and fills, with separate visual configurations for different sections of the application interface.

  5. Themes - The script checks the selected theme (fondo) and applies theme-specific visual configurations, including font color, slider background color (h0 and h1), values font color, and icons for turn on/off. This section allows users to customize the display’s look and feel.

  6. Text - The script displays navigation bar text based on the selected language (sys0). Language-specific text is set for each navigation bar segment and card titles within the main containers.

  7. Visual Elements - Visual elements are configured to highlight the currently selected navigation tab with a line and set the visibility of various components, such as values (value1 and value3), sliders (h0 and h1), and text components (t0 and t1).

  8. Values Background Color - The background color of the values components (value1 and value3) is set to match the selected theme’s color.

  9. Text and Background Color - The text (t0 and t1) and background color are configured based on the selected theme and font color.

Script Explanation

Let’s delve into the details of each section:

  1. Disabling Touch Components - Touch components h0 and h1 are disabled (tsw h0,0 and tsw h1,0) to prevent user interactions.

  2. Navigation Drawer and Navigation Bar - These sections define visual elements, such as circles, lines, and fills, to create the navigation drawer and navigation bar.

  3. Main Containers - Two main containers are defined with separate visual configurations. These containers may represent different sections of the application.

  4. Themes - The script checks the selected theme (fondo) and adjusts visual elements accordingly, including font color, slider background color, and icons.

  5. Text - Navigation bar text is set based on the selected language (sys0). Text is configured for different segments and card titles.

  6. Visual Elements - A line is used to highlight the selected navigation tab. Various components are made visible or hidden based on user interactions.

  7. Values Background Color - The background color of values components is set to match the selected theme.

  8. Text and Background Color - Text components’ text and background colors are configured based on theme and font color selections.

Conclusion

This documentation provides a comprehensive understanding of a Nextion display script that manages touch components, navigation elements, themes, and text based on user preferences. Developers can adapt and extend this script to create customized and interactive Nextion display interfaces for their applications.

Please note that this script serves as an example and should be integrated into a complete Nextion display project with appropriate user interactions, logic, and additional features as needed.
